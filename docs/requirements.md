# Требования к проекту

## Функциональные требования

- пользователь может просматривать образовательные материалы;
- материалы разделены по темам кибербезопасности;
- пользователь может пройти проверочные вопросы или тест;
- приложение отображается в браузере на компьютере и мобильном устройстве;
- структура проекта позволяет добавлять новые темы без полной переработки.

## Нефункциональные требования

- интерфейс должен быть понятным для неподготовленного пользователя;
- приложение должно работать стабильно в современных браузерах;
- тексты должны быть написаны простым языком;
- страницы должны открываться без заметных задержек;
- проект не должен хранить лишние персональные данные пользователей.

## Ограничения

- приложение носит образовательный характер;
- материалы не заменяют профессиональный аудит безопасности;
- рекомендации должны быть применимы для обычных пользователей;
- технические детали должны объясняться без чрезмерной сложности.

## Критерии готовности

- основные разделы приложения доступны из интерфейса;
- материалы по кибербезопасности структурированы по темам;
- тестовые или проверочные элементы работают корректно;
- проект запускается локально;
- документация описывает назначение, требования и запуск проекта.
