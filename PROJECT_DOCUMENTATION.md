# Cyber Arena Documentation

## Что изменилось

- Админская страница `/admin` оставлена только для локального доступа.
- Пункт меню админки убран из публичной навигации.
- Серверные мутации контента закрыты для внешних origin/host.
- Контент, прогресс, закладки, результаты и лидерборд работают через PostgreSQL.
- Добавлен отдельный demo-seed сценарий для повторного наполнения базы.

## Как теперь устроен контент

Источник стартового контента:

- `src/data/siteData.ts`

Клиентский слой получения и сохранения:

- `src/hooks/useContentState.tsx`

Утилиты для админки:

- `src/utils/contentAdmin.ts`

Админская страница:

- `src/pages/AdminPage.tsx`

Backend CRUD:

- `server/src/modules/content/content.routes.ts`
- `server/src/modules/content/content.repository.ts`

Публичные страницы читают тесты и материалы через API, а `siteData.ts` используется как fallback и как базовый набор для demo seed.

## Как использовать админку

1. Запустить API и frontend локально.
2. Открыть локальный адрес `http://localhost:4173/#/admin`.
3. Во вкладке `Тесты` можно создавать и редактировать сценарии.
4. Во вкладке `Материалы` можно создавать и редактировать учебные материалы.
5. Во вкладке `JSON / импорт` можно импортировать и экспортировать весь пакет контента.

Важно:

- через внешний туннель `/admin` не откроется
- внешние запросы на `POST/PUT/DELETE /api/content/*` и `PUT /api/content/bundle` получают `403`

## Как наполнить базу демонстрационными данными

После запуска PostgreSQL используйте:

```bash
npm run db:seed:demo
```

Скрипт:

- перезаполняет тесты и материалы
- создаёт демонстрационные команды и пользователей
- добавляет прогресс по материалам, закладки и попытки прохождения
- формирует живой лидерборд для публичных экранов

Файл скрипта:

- `server/src/db/seed-demo.ts`

## Формат JSON для импорта

Импорт ожидает объект такого вида:

```json
{
  "tests": [],
  "materials": []
}
```

Поле `tests` должно содержать массив объектов формата `CyberTest`, а `materials` массив объектов формата `MaterialItem`.

## Что важно понимать

- публичный сайт не обращается к `brainrivals.com` и не тянет чужие базы
- админка доступна только локально
- источник истины для демонстрационных данных теперь PostgreSQL
